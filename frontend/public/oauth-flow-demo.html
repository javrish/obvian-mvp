Step 3: User Saves Credentials

When the user clicks "Save Credentials":

// Frontend sends credentials to backend POST /api/oauth/credentials { "pluginId": "google", "userId": "demo-user", "clientId": "1072646854293-xxx.apps.googleusercontent.com", "clientSecret": "GOCSPX-xxxxx" } // Backend saves to ~/.obvian_oauth_credentials.properties google.demo-user.clientId=1072646854293-xxx.apps.googleusercontent.com google.demo-user.clientSecret=GOCSPX-xxxxx // Response { "success": true, "message": "OAuth credentials saved successfully" }
‚úÖ Credentials saved! The OAuth flow will now start automatically.

Step 4: OAuth Flow Starts Automatically

After credentials are saved, the OAuth flow initiates with the user's credentials:

// Frontend retries OAuth initiation POST /api/plugins/configure/google/oauth/initiate // Backend now uses saved credentials { "success": true, "authorizationUrl": "https://accounts.google.com/o/oauth2/v2/auth?client_id=USER_CLIENT_ID&...", "state": "xxx-xxx-xxx" }

A popup window opens with Google's OAuth consent screen using the user's Client ID.

Step 5: User Completes Authorization

The user logs into Google and grants permissions. After completion:

  • Access token is received and stored
  • Plugin status changes to Connected
  • User can now use Google Workspace features in their workflows
